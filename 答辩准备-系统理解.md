## 二、核心问题与答案

### Q1: 数据库里的数据是怎么进来的？

**答**: 有两条路径。

**路径一：SQL 初始化脚本**
- 文件位置：`zijin-server/src/main/resources/sql/init.sql`
- 首次部署时执行，直接向数据库插入测试数据（6个用户、3种角色、5个部门、9个专业、10个班级）

**路径二：用户通过前端页面操作**
- 以"新增部门"为例，完整数据流：

```
用户点击"新增部门" → Department.vue 弹窗表单 → 填写部门名称/编码
  → 点击保存 → handleSave() → api.post('/department', form)
  → Axios 发送 POST http://localhost:8088/api/department
    Body: {"deptName":"计算机学院","deptCode":"CS"}
  → DepartmentController.add() 接收请求
  → departmentService.save(dept)  // MyBatis-Plus 自动生成 INSERT SQL
  → INSERT INTO t_department (dept_name, dept_code, create_time) VALUES (...)
  → MySQL 写入磁盘 → 返回成功 → 前端刷新列表
```

**关键代码位置**:
| 层级 | 文件 | 关键行/方法 |
|------|------|-----------|
| 前端请求 | `src/api/index.js` | `api.post('/department', form)` |
| 前端页面 | `src/views/Department.vue` | `handleSave()` 函数 |
| 后端接口 | `controller/DepartmentController.java` | `@PostMapping` → `add()` |
| 业务层 | `service/DepartmentService.java` | 继承 `ServiceImpl`，自动 CRUD |
| 数据层 | `mapper/DepartmentMapper.java` | 继承 `BaseMapper<Department>` |
| 实体映射 | `entity/Department.java` | `@TableName("t_department")` |

### Q2: 移动客户端和服务器端是怎样通信的？

**答**: 前后端完全通过 **HTTP RESTful API** 通信，移动端和 PC 端共用同一套接口。

```
┌─────────────┐        HTTP Request         ┌──────────────┐
│  Vue 前端    │ ────  POST /api/login ────→ │  Spring Boot │
│  (5173端口)  │ ←───  {code:200, data:{}} ── │  (8088端口)   │
└─────────────┘        JSON Response         └──────────────┘
```

**具体通信方式**:
- 前端用 **Axios** 库发送 HTTP 请求
- 后端用 **Spring MVC** (`@RestController`) 接收并返回 JSON
- Vite 开发服务器配置了代理（`vite.config.js`），将 `/api` 请求转发到 `localhost:8088`
- 请求格式：`Content-Type: application/json`
- 响应格式：`{code: 200, msg: "success", data: {...}}`

**跨平台适配**（移动端同样走这套协议）：
- 前端使用响应式 CSS（`@media` 查询，断点 768px）
- Element Plus 组件自带移动端适配
- 移动端侧边栏变为抽屉式，触摸友好

### Q3: 是怎样保存 session 或者 cookie 的？

**答**: 本项目采用 **localStorage + 无状态 token 方案**，未使用传统 Session/Cookie。

> 说明：指导书要求降低复杂度、禁止安全校验，所以采用了最简单的 localStorage 方案。实际生产环境一般用 JWT Token。

**工作流程**:

```
1. 登录成功
   ↓
2. 后端返回用户信息 {id, username, realName, roleId, ...}
   ↓
3. 前端存入 localStorage:
   localStorage.setItem('zijin_user', JSON.stringify(userData))
   ↓
4. 后续每次路由跳转，router.beforeEach 检查 localStorage 是否有用户数据
   - 有 → 放行
   - 无 → 重定向到 /login
   ↓
5. 退出登录时: localStorage.removeItem('zijin_user')
```

**关键代码位置**:
| 功能 | 文件 | 位置 |
|------|------|------|
| 存储登录信息 | `src/views/Login.vue` | `handleLogin()` 方法内 |
| 路由拦截 | `src/router/index.js` | `router.beforeEach()` |
| 读取用户信息 | `src/views/Layout.vue` | `JSON.parse(localStorage.getItem(...))` |
| 退出清除 | `src/views/Layout.vue` | `handleLogout()` 方法 |

**浏览器查看**：F12 → Application → Local Storage → `zijin_user`

### Q4: 图表的数据是从数据库里面哪个表获取的？

**答**: 首页（Home.vue）的统计数据来源：

| 统计项 | 数据库表 | API 接口 | 查询方式 |
|--------|---------|----------|---------|
| 部门总数 | `t_department` | `GET /api/department` | 返回列表取 `.length` |
| 班级总数 | `t_class` | `GET /api/class` | 返回列表取 `.length` |

| 我的专业（学生） | `t_user` + `t_major` | `/api/major` | 根据 user.majorId 匹配 |

         │              │ 2  │ 软件工程  │ 1       │
         │              └────┴──────────┴─────────┘
         │                            │
         ▼                            │ major_id
    t_user (用户表)                    ▼
┌────┬───────┬─────┬─────────┬──────────┬──────────┐
│ id │username│ pwd │ role_id │ major_id │ class_id │
├────┼───────┼─────┼─────────┼──────────┼──────────┤

**核心关联**:
- `t_user.role_id` → `t_role.id` （用户属于哪个角色）
- `t_user.major_id` → `t_major.id` （用户属于哪个专业）
- `t_user.class_id` → `t_class.id` （用户属于哪个班级）
- `t_major.dept_id` → `t_department.id` （专业属于哪个部门）
- `t_class.major_id` → `t_major.id` （班级属于哪个专业）

## 六、角色权限控制原理

```
登录 → 后端返回用户 roleId (1=admin, 2=teacher, 3=student)
  → 前端存入 localStorage
  → Layout.vue 读取 roleId → 计算 roleCode
  → 菜单动态渲染：
    - admin:  人事管理 + 教务管理 + 系统管理
    - teacher: 仅教务管理
    - student: 仅首页
  → 路由 meta.roles 控制页面级权限（虽然目前主要通过菜单隐藏控制）
```

**三层权限**:
1. **菜单层**: Layout.vue 根据 `roleCode` 条件渲染菜单项
2. **路由层**: `router.beforeEach` 检查登录状态
3. **按钮层**: 班级管理中教师角色隐藏"新增/编辑/删除"按钮

## 七、常见答辩追问

### 1. 如果用户直接输入 URL /account 但角色是学生，会发生什么？

当前实现：菜单隐藏了入口，但如果直接输入 URL 仍可访问。**改进方案**：在 `router.beforeEach` 中添加角色检查。

### 2. 密码是明文存储的吗？为什么？

是的，`t_user.pwd` 存储明文密码 `123456`。指导书要求"禁止采用密码加密、安全校验等技术，采用最小技术栈"。

### 3. MyBatis-Plus 是什么？为什么用它？

MyBatis-Plus 是 MyBatis 的增强工具，`BaseMapper<T>` 提供内置 CRUD 方法（`insert`/`deleteById`/`updateById`/`selectList`），无需手写 SQL，大幅减少代码量。

### 4. 前端怎么知道登录用户是什么角色？

用户登录成功后，后端返回的用户对象包含 `roleId` 字段。前端将其存入 localStorage，Layout.vue 读取后映射为 `roleCode` 来控制菜单显示。

### 5. 移动端和 PC 端是怎么做到适配的？

CSS `@media (max-width: 767px)` 媒体查询 + Element Plus 响应式栅格系统（`el-row`/`el-col` 的 `xs`/`sm`/`md` 断点）。
